# V16.5 — SDR handoff/funil fix

Correções:

- Se o SDR já entregou o lead para vendedor/protótipo, ele não responde mais mensagens seguintes como "obrigado", "ok" ou novas mensagens curtas.
- O robô não repete a mensagem final de handoff.
- O SDR tenta mover a oportunidade no funil automaticamente:
  - quando identifica o negócio: tenta mover para "Contato feito";
  - quando entrega para vendedor/protótipo: tenta mover para "Diagnóstico" ou "Briefing recebido" conforme as etapas existentes no pipeline.
- A execução registra no output o `stageUpdate`, para diagnosticar se a mudança de etapa funcionou.

Observação:
- Para testar com o mesmo contato contaminado por versões antigas, apague o estado em `sdr_states` ou use um contato novo.
